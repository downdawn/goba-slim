# PGO 基线流程

PGO 只在有代表性负载和可重复证据时启用，不承诺固定 QPS。

1. 选择覆盖登录、认证、用户查询或实际热点的基准测试。
2. 在接近目标 CPU/OS 的环境运行 `task pgo:benchmark`，生成 `default.pgo`。
3. 记录 Go 版本、提交、硬件和基准命令，检查 Profile 不包含敏感标签或路径。
4. 使用 `task build:pgo` 构建并以同一负载比较延迟、CPU 和二进制体积。
5. 只有收益稳定且无回归时才提交 Profile；业务行为变化后重新评估。

当前仓库不提交默认 Profile，避免用不具代表性的单元负载固化优化方向。
