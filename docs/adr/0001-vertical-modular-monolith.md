# ADR 0001：垂直模块化单体

- 状态：已接受
- 日期：2026-07-13

## 决策

GoBA Slim 按业务能力组织模块，由 `internal/app` 显式装配。Gin 仅限 HTTP 边界，pgx/sqlc 和 go-redis 类型仅限基础设施适配器。OpenAPI 和 SQL 分别是 HTTP 与查询事实来源。

## 原因

该结构在保持单进程部署简单性的同时，限制框架类型扩散，支持独立测试和后续模块演进。完整 Clean Architecture 层数会增加首版仪式，横向 `controller/service/dao` 目录会削弱业务边界。

## 后果

跨模块依赖必须通过构造期窄接口；模块不能直接操作其他模块数据表。若未来拆分服务，需要在真实边界和可靠消息需求出现后另立 ADR。
