# ADR 0002：显式 Schema 管理

- 状态：已接受
- 日期：2026-07-13

## 决策

数据库变更以 `db/migrations` 中按序 SQL 文件交付，由显式 `goba db migrate` 命令在本地启动包装器或部署方的一次性 Job 中执行。常驻服务仅检查 PostgreSQL 和 Schema 版本，不执行运行时自动迁移。

## 原因

生产变更需要备份、审计、锁和回滚控制；把迁移绑定到每个服务实例会放大并发执行和权限风险。

## 后果

发布说明必须明确 SQL 变化。新环境可用 `goba db migrate` 初始化空库，已有 GoBA 数据库按版本执行增量 SQL；非空且未受管理的数据库拒绝接管。已应用迁移不修改，结构变化一律新增迁移文件。
